1.6 What We DON'T Access

For absolute clarity, SMPLTSK does NOT:

  • Read your task descriptions or comments

Think of SMPLTSK like a secure filing cabinet:

You put things in (photos, tasks, messages), the cabinet stores them safely (Firebase servers), only you and people you give keys to can open it (your chosen contacts), we don't look inside (SMPLTSK doesn't access your content), we just maintain the cabinet (technical maintenance and security).

Your photos, tasks, and messages are between you and the people you choose to share with. Period.

3. How We Use Firebase

Server Location: Your data is stored on Firebase servers located in Sydney, Australia.

Firebase Services We Use:

We utilize the full suite of Firebase services to ensure optimal app performance:

  • Firebase Authentication - Google and Apple Sign-In
  • Firebase Cloud Firestore - Task and user data storage
  • Firebase Cloud Storage - Image storage
  • Firebase Analytics - Usage patterns and feature adoption
  • Firebase Crashlytics - Crash reporting and debugging
  • Firebase Performance Monitoring - App speed and responsiveness
  • Firebase Cloud Messaging - Push notifications (you grant permission on signup)
  • Firebase Remote Config - Feature flags and A/B testing
  • Firebase App Distribution - Beta testing
  • Firebase Test Lab - Quality assurance

All Firebase services comply with Google's privacy standards and our data protection commitments.

4. SMPLTSK-Specific Features

4.1 Photo-Based Task Management

  • You upload photos which are stored in Firebase Cloud Storage (Sydney)
  • Visual markers (red for undone, green for done) are stored with each task
  • Task batches can contain multiple images

4.2 Folder Organization

Your tasks are automatically organized into folders:

  • Sent: Tasks you've sent to others
  • Received: Tasks others have sent to you
  • Mine: Personal tasks for yourself
  • Drafts: Unsent/incomplete task batches

We store folder metadata to maintain this organization.

4.3 Contact System

  • You can import contacts from your device's native contact list
  • Imported contacts are stored in our system to enable task sharing
  • We only access contact names and identifiers you explicitly import

4.4 In-App Messaging

  • Comments and messages between task senders and recipients are stored in Firebase
  • Messages are linked to specific tasks and retained as long as the task exists

4.5 Task Limitations

  • Once a task is sent, the photo, markers, and descriptions cannot be edited
  • Only completion status (red/green) can be changed
  • This ensures task integrity and clear accountability
